1. การยืนยันตัวตน (Authentication)

ระบบใช้ API Key ในการรักษาความปลอดภัย ทุกการร้องขอ (Request) ต้องแนบ Parameter api_key เสมอ

Parameter Type Required Description
api_key String Yes รหัสผ่านสำหรับเชื่อมต่อ (ติดต่อ Admin เพื่อขอรับ Key)

2. รายละเอียด Endpoints

2.1 ดึงสรุปยอดขายรายสาขา (Sales Summary)

ดึงยอดขายรวม แยกตามสาขา ในช่วงวันที่กำหนด (เหมาะสำหรับ Dashboard)

GET /api_sales_summary.php

Parameters:

Parameter Required Default Description
api_key Yes - API Key
date_from No Today วันที่เริ่มต้น (YYYY-MM-DD)
date_to No Today วันที่สิ้นสุด (YYYY-MM-DD)

Example Response:

{
  "status": "success",
  "period": { "from": "2026-01-01", "to": "2026-01-01" },
  "count": 2,
  "data": [
    {
      "store_code": "001",
      "store_name": "สาขาหลัก",
      "bill_count": 10,
      "total_qty": 50,
      "total_sales": 15000.00
    },
    {
      "store_code": "002",
      "store_name": "สาขาสยาม",
      "bill_count": 5,
      "total_qty": 12,
      "total_sales": 8500.00
    }
  ]
}

2.2 ดึงรายละเอียดบิล (Transaction Detail)

ตรวจสอบรายละเอียดบิล หรือดึงรายการบิลทั้งหมดใน 1 วัน

GET /api_transaction.php

Parameters:

Parameter Required Description
api_key Yes API Key
ref Conditional เลขที่บิลที่ต้องการดูสินค้าภายใน (เช่น SO-202601-001)
date Conditional วันที่ต้องการดูรายชื่อบิลทั้งหมด (YYYY-MM-DD)
หมายเหตุ: ต้องระบุอย่างใดอย่างหนึ่งระหว่าง ref หรือ date

Example Response (By Ref):

{
  "status": "success",
  "data": [
    {
      "internal_ref": "SO-202601-001",
      "line_barcode": "885xxxx",
      "item_description": "สินค้า A",
      "qty": 2,
      "base_price": 100.00,
      "tax_incl_total": 200.00
    }
  ]
}

2.3 ดึงข้อมูลดิบ (Raw Data Export)

ดึงข้อมูลดิบทุก Transaction อย่างละเอียด (เหมาะสำหรับ Data Warehouse/Backup)

GET /api_daily_sales_raw.php

Parameters:

Parameter Required Default Description
api_key Yes - API Key
date_from No Today วันที่เริ่มต้น
page No 1 หน้าของข้อมูล (ระบบแบ่งหน้าทีละ 1,000 รายการ)

Example Response:

{
  "status": "success",
  "pagination": {
    "current_page": 1,
    "per_page": 1000,
    "total_pages": 5,
    "total_records": 4500
  },
  "data": [
    { "internal_ref": "SO-001", "sale_date": "2026-01-01", ... }
  ]
}

3. รหัสข้อผิดพลาด (Error Codes)

Status Code Message ความหมาย/วิธีแก้ไข
401 API Key is required ไม่ได้ส่งค่า api_key มาใน Request
401 Invalid or Inactive API Key API Key ไม่ถูกต้อง หรือถูกระงับการใช้งาน
400 Please provide 'ref' or 'date' ส่ง Parameter ไม่ครบ (สำหรับ api_transaction)
500 Database Connection Failed ระบบฐานข้อมูลขัดข้อง

4. ตัวอย่าง Code การเชื่อมต่อ

Python

import requests

url = "https://www.weedjai.com/api/api_sales_summary.php"
params = {
    "api_key": "YOUR_API_KEY",
    "date_from": "2026-01-01",
    "date_to": "2026-01-31"
}

try:
    response = requests.get(url, params=params)
    if response.status_code == 200:
        data = response.json()
        print(f"Total Records: {data['count']}")
    else:
        print("Error:", response.text)
except Exception as e:
    print("Connection failed:", e)

PHP (cURL)

<?php
$url = "https://www.weedjai.com/api/api_daily_sales_raw.php?api_key=YOUR_KEY&page=1";
$ch = curl_init($url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = curl_exec($ch);
curl_close($ch);

$data = json_decode($response, true);
print_r($data);
?>
